Cytron Tutorial WI Summary

This summary captures the operational intent of the tutorial WI without exposing internal credentials or sensitive account instructions.

What the WI standardizes

  • Tutorial setup fields such as title, pitch, SEO URL, target audience, project level, related posts, related products, and store visibility.
  • Thumbnail expectations: 16:9 ratio, preferred 1280×720, clear hardware/output image, and optimized file size.
  • Content flow: introduction, requirements, step-by-step explanation, images/screenshots, code or command lines, and end-of-post community banner.
  • Writing quality: correct grammar, simple wording, beginner-friendly explanation, and clearly visible images.

What Khairul is building on top of it

  • A more structured writer-facing template to make new and legacy tutorials easier to standardize.
  • Clearer expectations for component links, system diagrams, software setup, testing, troubleshooting, and next-step guidance.
  • A repeatable publishing standard that can be used by contributors and trainees more consistently.
